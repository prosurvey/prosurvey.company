# Гид По Редактированию Контента

Этот проект сделан так, чтобы основной контент можно было менять без правки компонентов.

## Что Где Менять

- Главная страница: `src/content/home.yml`
- Политика конфиденциальности: `src/pages/privacypol.md`
- Условия использования: `src/pages/termofuse.md`
- Изображения: папка `public/images/` (в `home.yml` указываются пути вида `/images/имя-файла.png`)

## Важные Блоки В `home.yml`

- `site`: заголовок сайта, описание для SEO.
- `nav.header`: пункты меню и кнопка в шапке.
- `sections.hero`, `sections.about`, `sections.services`: тексты основных блоков.
- `sections.request`: тексты формы заявки.
- `sections.request.messages.requiredField`: текст ошибки у обязательного поля.
- `sections.request.messages.invalidForm`: текст при невалидной отправке формы.
- `sections.request.messages.success`: текст после успешной отправки (пока заглушка).
- `sections.contacts`: почта, телефон, соцсети.
- `footer`: копирайт и ссылки в подвале.
- `cookie`: текст cookie-баннера, ссылка и кнопка `acceptText`.

## Правила Для YAML

- Используйте пробелы, не табы.
- Соблюдайте отступы: обычно 2 пробела на уровень.
- Строки с `:` внутри лучше брать в кавычки.
- Массивы начинаются с `-`.
- Если текст в несколько строк, используйте `|` (как уже сделано в файле).

## Быстрая Проверка После Правок

Из папки `new_design`:

```sh
npm run build
```

Если структура YAML нарушена или поле пропущено, сборка покажет ошибку с путем до проблемного поля.
